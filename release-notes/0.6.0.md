## GridSense 0.6.0

- Clearing cells no longer deletes properties: emptying a cell (Delete, find & replace, clearing a column) keeps the property with an empty value. Only explicit delete actions remove keys.
- Fixed: the first and last rows of the grid couldn't really be edited (typing took one character, double-click did nothing) — the sticky top draft row was overlapping them. The top draft now scrolls with the grid and looks identical to the bottom one.
- Right-click a row number (or the note link): duplicate the note (\"name (copy)\") or rename it — renames update links vault-wide.
- Add column: header right-click or the columns manager — adds a property (empty) to every note in the grid, undoable.
- Delete column: header right-click, with a confirmation warning that it removes the property and values from all notes in the grid — undoable while the tab is open, and always in the edit log.
- Formulas from the note properties strip: add/edit folder formulas right where you edit a note's properties; they render computed (ƒ, never written to notes) and appear as columns on the folder's grid automatically.
